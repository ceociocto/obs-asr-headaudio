# ⚠️ 试验性方案: 快速响应优化

> **这是试验性方案，用于探索低延迟 TTS 响应技术**
>
> **正式稳定版本**: commit `f1caaad` (feat: qwen tts stream and llm stream)
>
> 本方案在正式版基础上实现了多层延迟优化，但目前处于试验阶段，仅供参考和测试。

## 三层优化

```
┌─────────────────────────────────────────────────────────────────┐
│  用户提问                                                        │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 1: 填充音频 (<100ms)                                     │
│  立即播放 "Let me see..." 等预录制音频                           │
├─────────────────────────────────────────────────────────────────┤
│  Layer 2: 预测性开头 (200-500ms)                                │
│  根据问题类型预测回答开头，提前 TTS                             │
├─────────────────────────────────────────────────────────────────┤
│  Layer 3: 激进短语分割 (提前1-2s)                                │
│  不等完整句子，按逗号/连词分割                                   │
└─────────────────────────────────────────────────────────────────┘
```

### Layer 1: 填充音频

**原理**: 立即发送预录制的短音频

**效果**: 用户感知延迟从 2-4秒 降至 <0.1秒

**文件**: `filler_audio/*.wav`

| 音频 | 内容 | 时长 |
|------|------|------|
| `let_me_see.wav` | "Let me see." | 1.36s |
| `let_me_check.wav` | "Let me check." | 1.44s |
| `thinking.wav` | "Hmm." | 2.64s |
| `one_moment.wav` | "One moment." | 1.36s |
| `give_me_sec.wav` | "Just a second." | 1.52s |

### Layer 2: 预测性开头 TTS

**原理**: 根据问题类型预测回答开头，提前进行 TTS

**准确率**: 50-95% (视问题类型)

| 问题类型 | 关键词 | 预测开头 | 准确率 |
|---------|-------|---------|-------|
| 问候 | hello, hi, 你好 | "Hello!" / "你好！" | ~95% |
| 时间 | time, 几点 | "让我查查时间" | ~80% |
| 天气 | weather, 天气 | "让我查查天气" | ~70% |

### Layer 3: 激进短语分割

**原理**: 不等待完整句子，按逗号、连词分割

**效果**: 首个音频片段提前 1-2 秒开始

**分割规则**:
- 逗号、分号、冒号后分割
- 连词前分割 (and, but, or, because...)
- 超过 40 字符强制分割

## 使用方法

### 1. 生成填充音频 (首次运行)

```bash
python generate_filler_audio.py
```

### 2. 启动服务器

```bash
python qwen3_tts_fast_server.py
```

服务器启动后会显示:
```
✓ Filler audio: 5 loaded
```

### 3. 打开演示页面

```bash
open qwen3-tts-filler-demo.html
```

在演示页面中可以开关填充音频功能，对比效果。

## 已知问题

1. **预测性 TTS 可能不准确**: 某些问题类型的预测可能与实际回答不符
2. **短语分割可能破坏语意**: 激进分割可能在某些情况下影响语义连贯性
3. **填充音频可能重复**: 如果实际回答开头与填充音频相似，可能造成重复

## 与正式版兼容性

本试验版与正式版 **完全兼容**:
- 使用相同的 WebSocket 协议
- 使用相同的 LLM API
- 可以在正式版基础上启用/禁用各层优化

客户端通过以下参数控制优化:
- `use_filler`: 是否使用填充音频
- `use_predictive`: 是否使用预测性 TTS
- `use_phrase_split`: 是否使用短语分割
